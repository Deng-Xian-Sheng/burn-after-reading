这个web应用的界面风格和google搜索差不多：看上去简洁、科技。

其中，`/`这个页面，用户可以上传或拖放或从剪切板粘贴图片，上传完成之后会得到一个链接，链接的域名后面的部分类似于`/此处是短随机字符串#此处是解密密钥`，第一次访问这个链接会显示用户上传的那张图片（没有UI，返回一个html用于解密图片，然后在浏览器上显示图片），第二次访问这个链接会显示一个带有“404”大字的页面，http状态也是404，“404”大字下面有一行文本：“有内鬼，中止交易！”，这行文本用glitch风格呈现。

图片大小限制为5MB，图片在第一次访问完成之后就被删除了，如果未被访问，图片保留时间限制为24小时，超过之后会被删除。

在第一次访问返回的HTML中添加一些约束，例如无法右键保存图片、无法按住图片保存到相册……
我们可能无法限制用户截图什么的，但尽可能的加些约束。

我设计了一种端到端的加密方法，确保后台也看不到用户上传的图片：
- 上传前在浏览器把图片加密
- 服务端只存密文
- 访问时服务端把密文发回浏览器
- 浏览器端解密并展示图片
- 链接`/id#key`，id部分用于索引，key是解密密钥，只存在浏览器端，不会被 HTTP 请求发给服务端
- 为了防止别人只拿到 id 就能把密文取走并销毁（DoS），在前端上传图片时，把key用sha256哈希并传给服务端存起来，读取时前端拿到`#`后面的key再次sha256并传给服务端，服务端只比对sha256后的key，仍然不知道 key，也无法解密图片
